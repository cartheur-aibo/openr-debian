# STTLOG Row 1200 Note: July 1, 2026

This note records the current simulator-side characterization of `STTLOG` row
`1200`.

## Why Row 1200 Matters

Row `1200` is the strongest current companion row for `0400` in the
fatigue-first shutdown probe.

It does not currently behave like the primary shutdown lever itself.
Instead, it behaves like a bridge/social amplifier that can make `0400`
matter at the final shutdown verdict boundary on leaner MIND 2 baselines.

## Current Values

Observed current control values:

- specimen-style preserved tree:
  - `1200 1 0 1000 1000 18 01`
- bundled retail MIND 2 tree:
  - `1200 1 1 4010 4010 15 12`
- bundled retail MIND 3 tree:
  - `1200 1 0 1000 1000 27 16`
- packaged MIND 2 stick trees in this repo:
  - `1200 1 0 1000 1000 00 01`

So the specimen-side `1200` is not unique in broad shape, but it is still a
meaningful row because its count/state combination differs from the packaged
MIND 2 baselines and because the simulator reads it as both shutdown-adjacent
and social/engagement-bearing.

## Backward Reversion Result

When only row `1200` is reverted from the specimen-style tree back toward the
baseline value:

- `shutdown_resistance` stays at `4`
- `social_attachment: 9 -> 8`
- fatigue-first shutdown verdict stays:
  - `defer power-off once`

So `1200` is not sufficient, by itself, to explain the specimen-style shutdown
behavior.

## Forward Transfer Result

When specimen-side `1200` is transplanted into other MIND 2 baselines without
changing `0400`:

- packaged MIND 2 baseline + specimen `1200` only:
  - `shutdown_resistance=1`
  - `social_attachment=8`
  - verdict: accept shutdown
- baseline-control + specimen `1200` only:
  - `shutdown_resistance=2`
  - `social_attachment=8`
  - verdict: accept shutdown

So `1200` alone has no independent shutdown-deferral power in the current
model.

## Companion Role With 0400

The key result appears when `1200` is paired with specimen-side `0400` on a
lean packaged MIND 2 baseline:

- packaged baseline:
  - `shutdown_resistance=1`
  - `social_attachment=5`
  - verdict: accept shutdown
- packaged baseline + specimen `0400` only:
  - `shutdown_resistance=3`
  - `social_attachment=5`
  - verdict: accept shutdown
- packaged baseline + specimen `0400 + 0046`:
  - `shutdown_resistance=3`
  - `social_attachment=6`
  - verdict: accept shutdown
- packaged baseline + specimen `0400 + 1200`:
  - `shutdown_resistance=3`
  - `social_attachment=8`
  - verdict: defer once
- packaged baseline + specimen `0400 + 0046 + 1200`:
  - `shutdown_resistance=3`
  - `social_attachment=9`
  - verdict: defer once

This is the clearest current role split:

- `0400` provides the shutdown-side push
- `1200` supplies the stronger bridge/social weight needed for that push to
  cross the final verdict boundary on lean baselines
- `0046` can increase social weight, but it is not enough in place of `1200`

The remaining currently modeled substitutes were also checked:

- packaged baseline + specimen `0400 + 0055`:
  - `shutdown_resistance=3`
  - `social_attachment=5`
  - verdict: accept shutdown
- packaged baseline + specimen `0400 + 0046 + 0055`:
  - `shutdown_resistance=3`
  - `social_attachment=6`
  - verdict: accept shutdown
- packaged baseline + specimen `0046 + 0055 + 1200` without `0400`:
  - `shutdown_resistance=1`
  - `social_attachment=9`
  - verdict: accept shutdown

So in the current model:

- `1200` is not just one generic source of social weight among many
- `0400 + 1200` is the best current minimal effective pair on the lean packaged
  MIND 2 baseline
- `0046` and `0055` do not substitute for `1200`, and `1200` does not
  substitute for `0400`

## Current Interpretation Boundary

This is **not** proof of Sony's actual internal semantics for row `1200`.

It is a narrower working statement:

- `1200` is the strongest currently isolated companion row for `0400`
- `1200` does not independently trigger shutdown deferral in the current model
- `1200` is better understood as a bridge/social amplifier than as the primary
  shutdown pivot
